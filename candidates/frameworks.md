    每年一次的360度反馈，但不匿名、不评分、不与升职加薪挂钩。使用"开始/停止/继续"三类反馈格式。鼓励员工给尽可能多的同事（不仅是直接上下级）提供反馈。关键进化：从最初的匿名模式演变为全员具名——在坦诚文化成熟后，匿名反而降低了反馈质量和可操作性。领导者与团队分享自己收到的反馈（尤其是批评），形成"反向负责制"。
  tags: [反馈, 评估, 组织学习]

- id: f25

- id: f26
  title: 情景管理而非控制管理 (Context Not Control)
  type: framework
  source_chapter: Ch9
  source_quote: |
    "不要告诉员工应该做什么，而是应该通过讨论来设置情景，达成一致，最后让他们自己去做出正确的决定。"
  summary: |
    两种管理模式的根本区别：控制型管理由老板做决策并自上而下传达，用流程控制代替直接监督；情景管理将一切信息透明化，团队成员在不受监管的情况下自行决策。情景管理需要四个条件：(1)高人才密度；(2)创新性目标（而非防范错误）；(3)松散耦合的体制；(4)全组织认识一致。当员工做了蠢事，管理者不应指责，而应反思自己在情景设定上犯了什么错。
  tags: [管理哲学, 领导力, 决策]

- id: f27
  title: 认识一致，松散耦合 (Aligned and Loosely Coupled)
  type: framework
  source_chapter: Ch9
  source_quote: |
    "认识一致，松散耦合。"
  summary: |
    借用软件工程概念描述组织架构。松散耦合：组织各部门像独立模块，可独立运作和变更。紧密耦合：各部门错综复杂地交织在一起，修改任何部分都需要重建基础。网飞选择松散耦合——部门经理甚至员工本人有权自行做决定，前提是问题不会波及其他部门。但松散耦合的前提是"认识一致"——老板和员工必须就目标和战略达成共识。没有一致性的松散耦合等于各自为政。
  tags: [组织设计, 系统架构, 战略对齐]

- id: f28
  title: 树形情景管理法 (Tree-Shape Context Setting)
  type: framework
  source_chapter: Ch9
  source_quote: |
    "首席执行官就是树的根部，而伸展开去的知情指挥则位于树枝顶端，负责各项具体决策的制定。"
  summary: |
    替代金字塔决策的树形模型。CEO（树根）设定第一层情景（如全球化战略），高管（树干）在CEO基础上设定第二层情景（如大胆下注），中层管理者（粗枝）进一步设定情景（如聚焦动画），基层知情指挥（小枝丫）在所有层次情景的指导下自主做出最终决策。情景由根部逐层传递至枝丫，但决策权在枝丫而非根部。每一层管理者只做情景设定，不做决策审批。
  tags: [管理架构, 情景管理, 决策权分配]

  title: 跨文化反馈适应法 (Cross-Cultural Feedback Adaptation)
  type: framework
  source_chapter: Ch10

- id: f30
  title: 升格语/降格语反馈调整法 (Upgrader/Downgrader Adjustment)
  type: framework
  source_chapter: Ch10
  source_quote: |
    "直接文化倾向于使用'升格语'……间接文化在给予负面反馈时则更多使用'降格语'。"
  summary: |
    跨文化反馈的具体语言工具。直接文化（如荷兰、美国）在负面反馈中使用"升格语"（"绝对""完全""强烈"），使反馈更强势。间接文化（如日本、新加坡）使用"降格语"（"稍微""有几分""可能""略微"），软化批评语气。跨文化沟通时，直接文化的人需要在反馈前增加正面肯定，用建议而非命令的口吻，并考虑添加表情等柔和信号。间接文化的人则需要理解，直接反馈不是攻击。

- id: f32

  type: framework
  source_chapter: Ch8
  source_quote: |
    "360度公开反馈引发了一系列有价值的讨论……形成了一种'反向负责制'。"
  summary: |
    通过360度反馈和面对面反馈机制，形成员工对上级的常态化问责。领导者公开分享自己收到的批评性反馈（如里德分享360度评估结果），鼓励下属据此监督和反馈上级的行为。这不是自上而下的管理，而是由反馈文化驱动的自下而上约束力。领导者不命令员工怎么做，但团队成员之间形成相互反馈的网络，确保每个人（包括领导）都对自己的行为负责。

- id: f33
  title: 防范错误 vs 勇于创新选择框架 (Error-Prevention vs Innovation Choice)
  type: framework
  source_chapter: Ch9
  source_quote: |
    "如果重点是防范错误，控制型管理是最好的做法……但如果公司的目标重在创新，那错误并不是主要的风险。"
  summary: |
    选择管理模式的关键决策框架。先回答两个问题：(1)公司员工属于哪个层次的人才？高绩效→情景管理更优；还在成长→需要控制管理。(2)目标是防范错误还是创新？医疗、核能、航空等零容忍行业→控制型管理；创意、娱乐、零售等创新驱动行业→情景管理。第三，组织架构是紧密耦合还是松散耦合？紧密耦合型公司很难推行情景管理。最后，团队是否认识一致？没有一致性的松散耦合等于各自为政。
